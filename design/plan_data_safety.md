# Play Store データ安全フォーム 設計

## 概要

Google Play Console の「データの安全性」セクションへの回答内容。
対象: LearnApp (com.yanheng.learnapp)

---

## 1. データ収集・共有の概要

| 質問 | 回答 |
|---|---|
| アプリがデータを収集または共有しているか | **はい** |
| すべてのデータが暗号化されているか | **はい**（HTTPS/TLS）|
| ユーザーがデータの削除をリクエストできるか | **はい**（アカウント削除機能）|

---

## 2. 収集するデータの種類と申告内容

### 2-1. 個人情報

| データの種類 | 項目 | 収集 | 共有 | 目的 | 処理 |
|---|---|---|---|---|---|
| 個人情報 | メールアドレス | ✅ | ❌ | アカウント管理 | サーバー送信・保存 |
| 個人情報 | パスワード（ハッシュ化） | ✅ | ❌ | 認証 | サーバー送信・保存 |

### 2-2. アプリのアクティビティ

| データの種類 | 項目 | 収集 | 共有 | 目的 | 処理 |
|---|---|---|---|---|---|
| アプリのアクティビティ | 学習タスク情報（タスク名・科目・曜日設定）| ✅ | ❌ | アプリ機能 | サーバー送信・保存 |
| アプリのアクティビティ | 日々の学習記録（完了状況・学習時間）| ✅ | ❌ | アプリ機能 | サーバー送信・保存 |
| アプリのアクティビティ | 集計データ（月次サマリー）| ✅ | ❌ | アプリ機能 | サーバー送信・保存 |

### 2-3. ユーザーコンテンツ

| データの種類 | 項目 | 収集 | 共有 | 目的 | 処理 |
|---|---|---|---|---|---|
| ユーザーコンテンツ | 子どもの名前・学年 | ✅ | ❌ | アプリ機能 | サーバー送信・保存 |

---

## 3. データの取り扱い詳細

### 保存・転送

| 項目 | 内容 |
|---|---|
| **サーバー所在地** | 日本 |
| **転送の暗号化** | HTTPS（TLS）を使用 |
| **ローカル保存** | JWT トークンのみ（Android DataStore）|
| **第三者への共有** | なし（広告・分析 SDK は一切不使用）|

### データ保持・削除

| 項目 | 内容 |
|---|---|
| **アカウント削除後のデータ** | 即時削除 |
| **削除リクエスト手段** | アプリ内（アカウント削除機能）※実装予定 |

---

## 4. Play Console 入力手順

### ステップ 1: データ安全性セクションを開く

```
Play Console
→ アプリを選択
→ 左メニュー「ポリシー」>「データの安全性」
→「開始」ボタン
```

### ステップ 2: データ収集の概要

```
Q: このアプリはユーザーのデータを収集または共有しますか？
A: はい

Q: アプリで送信するすべてのユーザーデータは転送中に暗号化されていますか？
A: はい

Q: ユーザーはデータの削除をリクエストできますか？
A: はい
```

### ステップ 3: データの種類を選択

チェックする項目:
- [x] **個人情報** → メールアドレス
- [x] **アプリのアクティビティ** → アプリのインタラクション
- [x] **ユーザーコンテンツ** → その他のユーザー生成コンテンツ

チェックしない項目（使用していない）:
- [ ] 位置情報
- [ ] 財務情報
- [ ] 健康とフィットネス
- [ ] 写真と動画
- [ ] 音声ファイル
- [ ] ファイルとドキュメント
- [ ] 連絡先
- [ ] 通話とSMS
- [ ] デバイスまたはその他のID

### ステップ 4: 各データの詳細入力

**メールアドレス**
```
収集: はい
共有: いいえ
必須 or 任意: 必須
処理の目的: アカウント管理
```

**子どもの名前・学年**
```
収集: はい
共有: いいえ
必須 or 任意: 名前は必須 / 学年は任意
処理の目的: アプリの機能
```

**学習記録（タスク・日々の記録・集計）**
```
収集: はい
共有: いいえ
必須 or 任意: 任意
処理の目的: アプリの機能
```

---

## 5. 事前確認チェックリスト

フォーム記入前に以下を確認する:

- [ ] アカウント削除機能が実装済みである（Play Store 審査要件）
- [ ] プライバシーポリシーの URL が用意されている
- [ ] バックエンドが HTTPS 通信のみ使用している
- [ ] 第三者 SDK（Firebase・AdMob 等）が未使用であることを確認

---

## 6. プライバシーポリシー URL

Play Console へのフォーム提出時にプライバシーポリシーの URL が必須。

現状: アプリ内に `PrivacyPolicyScreen.kt` が実装済みだが、**外部 URL（Web ページ）**が別途必要。

対応案:
- GitHub Pages で公開（無料）
- Google Sites で公開（無料）
